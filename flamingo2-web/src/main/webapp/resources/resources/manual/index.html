설치

설치에서는 Cloudera CDH, Pivotal HD, Hortonworks HDP를 제외하고 순수 Apache Hadoop 배포판에 대해서 설치를 설명합니다. Cloudera CDH, Pivotal HD, Hortonworks HDP의 경우 프로비저닝을 담당하는 Cloudera Manager, Ambari, Command Center가 제공되므로 본 문서에서는 별도로 논하지 않습니다.

Flamingo 2

Flamingo 2가 동작하기 위해서는 아래의 컴포넌트가 설치되어야 합니다. Apache Hadoop의 경우 Cloudera CDH, Pivotal HD, Hortonworks HDP에서 자동으로 구성되므로 여기에서는 순수 Apache Hadoop 배포판을 제외하고는 논하지 않습니다.

  • Chrome (테스트중: Internet Explorer 9 이상, Firefox, Safari)

  • Apache Hadoop 2.3 이상 (Apache Hive 1.0 권장함. 로그 표시가 1.0 이상 부터 지원)

  • JDK 1.7 이상

  • MySQL Server 5.x 이상 (MySQL 5.1 권장)

  • Flamingo 2 Web Application

  • Flamingo 2 MapReduce Job Agent

  • Flamingo 2 Namenode Agent

  • Flamingo 2 Resource Manager Agent

  • Flamingo 2 Collector

  • Flamingo 2 System Agent

  • node.js

  • R & RStudio Server (선택사항)

  • Pivotal HAWQ (선택사항)

Note
 상기 버전 이외에 환경에서 Flamingo를 동작시키거나 사용해야 하는 경우 기술지원(sales@cloudine.co.kr)을 요청하십시오.

JDK 설치

Flamingo 2는 JDK 1.7을 기반으로 개발되어 있으므로 Linux 에서 JDK 1.7 설치는 필수 사항입니다. 우분투에서는 다음의 커맨드로 JDK 1.7을 설치할 수 있습니다.

# add-apt-repository ppa:webupd8team/java
# apt-get update
# apt-get install oracle-java7-installer
# export JAVA_HOME=/usr/lib/jvm/java-7-oracle

CentOS, Ubuntu의 패키지 관리자를 이용하지 않고 직접 설치하고자 하는 경우 http://java.oracle.com 에서 JDK 1.7을 다운로드하여 다음과 같이 설치할 수 있습니다.

# cd /usr/local
# tar zxvf jdk-7u51-linux-x64.gz

JDK 1.7을 다운로드 및 압축 출기를 한 후 다음과 같이 사용자 환경에서 JDK를 사용할 수 있도록 설정합니다. 전체 사용자에 적용하지 않고 해당 사용자에게만 적용하고자 하는 경우 ~/.profile 또는 ~/.bashrc 파일에 다음을 설정하도록 합니다.

# vi /etc/profile
export JAVA_HOME=/usr/local/jdk1.7
export PATH=$JAVA_HOME/bin:$PATH

# source /etc/profile

MySQL Server 5 설치

Flamingo가 동작하기 위해서 필요한 데이터베이스는 MySQL이며 MySQL 5.x 이상 버전을 충족해야 합니다. 다음은 CentOS에서 MySQL을 설치하는 커맨드입니다. 이 커맨드는 반드시 root 권한으로 실행해야 합니다.

# yum -y install mysql-server

또한 Flamingo는 multi-bytes로 구성된 언어(예; CJK)를 저장하므로 MySQL은 기본으로 UTF-8을 지원해야 합니다. 하지만 CentOS에서 기본으로 설치되는 MySQL은 Latin 1으로 설정이 됩니다. 이를 변경하기 위해서는 MySQL 인코딩 변경 페이지를 참고해서 변경해야 합니다.

Note
 CentOS와 Ubuntu에서 사용하는 MySQL 버전이 서로 상이하기 때문에 UTF-8 설정이 모두 동일하게 적용되는 것은 아닙니다. 설정이 정상적으로 반영되지 않으면 MySQL Server가 시작할 수 없습니다.

Flamingo에서 사용하는 Quartz Job Scheduler는 primary key로 두개 이상의 컬럼을 조합해서 사용합니다. 따라서 이 길이가 MySQL에서 정하는 primary key의 길이를 초과하면 Specified key was too long 에러가 발생합니다. 이 경우 Quartz Job Scheduler용 MySQL 데이터베이스를 Latin 1으로 분리해서 사용하거나 또는 Quartz Job Scheduler의 MySQL 스키마에서 primary key의 길이를 최대 길이 이내로 수동 조정하도록 합니다.

Flamingo 실행 권한

Flamingo는 모든 작업에 대해서 Workflow Designer가 권한을 얻기 위해서는 proxy user 또는 mapred 계정으로 Flamingo를 실행시키도록 합니다.

Apache Hive

Note
 Flmaingo의 Apache Hive 관련 기능이 동작하려면 Apache Hive 1.0을 권장합니다. Apache Hive 1.0에 대한 자세한 변경 정보는 Apache Hive 1.0 Release Note을 참고하십시오.

Apache Hive를 구성하는 구성요소는 다음과 같습니다.

  • Hive Server 2 (기본 10000 포트 사용)

  • Hive Metastore (기본 9083 포트 사용)

Hive Server 2, Hive Metastore가 제대로 동작하기 위해서는 Apache Hive의 설정 파일은 <HIVE_HOME>/conf/hive-site.xml 파일에 Hive Metastore 정보가 정상적으로 설정되어 있어야 합니다. 또한 Metastore를 구성하기 위해서 MySQL 또는 PostgreSQL 등을 설정해야 합니다. 본 문서에서는 MySQL에 대해서만 설명합니다.

Note
 Cloudera CDH, Pivotal HD, Hortonworks HDP 배포판의 경우 자동으로 Apache Hive를 설치하므로 Cloudera CDH, Pivotal HD, Hortonworks HDP 배포판을 사용하는 사용자의 경우 본 내용을 넘어가도록 합니다.

MySQL 데이터베이스 생성하기

Flamingo의 Apache Hive 편집기를 포함하여 Apache Hive를 사용하기 위해서는 Metastore Server와 Hive Server 2가 설치되어 있어야 합니다. 이를 위해서 가장 먼저 Hive Metastore를 위한 MySQL 계정을 생성합니다. mysql 커맨드를 이용하여 로그인한 후 다음의 커맨드를 순서대로 입력합니다. 이렇게 하면 hive/hive인 새로운 MySQL 사용자가 생성되며 모든 권한을 부여하게 됩니다.

mysql> CREATE DATABASE hive;
mysql> CREATE USER 'hive'@'localhost' IDENTIFIED BY 'hive';
mysql> GRANT ALL PRIVILEGES ON * . * TO 'hive'@'localhost';
mysql> FLUSH PRIVILEGES;

Hive Metastore 설정

이제 마지막으로 <HIVE_HOME>/conf/hive-site.xml 파일을 다음과 같이 수정합니다.

<?xml version="1.0"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>
  <property>
    <name>javax.jdo.option.ConnectionUserName</name>
    <value>hive</value>
  </property>

  <property>
    <name>javax.jdo.option.ConnectionPassword</name>
    <value>hive</value>
  </property>

  <property>
    <name>hive.server2.thrift.port</name>
    <value>10000</value>
  </property>

  <property>
    <name>javax.jdo.option.ConnectionURL</name>
    <value>jdbc:mysql://localhost:3306/hive</value>
  </property>
</configuration>

Hive Server 2 프로세스 시작

이제 정상 동작하므로 Hive Server 2를 다음의 커맨드로 백그라운드 실행합니다. 이렇게 실행하면 10000 포트로 Hive Server 2가 동작하게 되며 Thrift API 또는 JDBC를 이용하여 Hive Server 2에 Hive Query 실행을 요청할 수 있게 됩니다.

# cd <HIVE_HOME>/bin
# nohup hiveserver2 &

Hive Metastore 프로세스 시작

Hive Server 2 뿐만 아니라 Hive 편집기에서 테이블 및 데이터베이스를 관리하려면 Hive Metastore Server가 동작해야 합니다. 이를 위해서 다음의 커맨드를 사용하여 Metastore Server를 실행할 수 있습니다. 이렇게 하면 9083 포트로 Hive Metastore Server가 동작하게 됩니다.

# cd <HIVE_HOME>/bin
# hive --service metastore &

설치시 에러 발생 사례

Specified key was too long 에러

MySQL이 UTF-8로 설정되어 있는 경우 Hive의 Metastore를 MySQL로 설정하게 되면 com.mysql.jdbc.exceptions.jdbc4.MySQLSyntaxErrorException: Specified key was too long; max key length is 767 bytes 에러가 발생할 수 있습니다. 이 경우 Latin 1으로 변경해야 합니다.

mysql> alter database hive character set latin1;

Flamingo Agent

Flamingo 2에는 Hadoop EcoSystem을 구성하는 각종 컴포넌트를 모니터링하고 고급 기능을 제공하기 위해서 JVM 상에서 동작하는 Agent를 각 컴포넌트에 설치합니다. Flamingo Engine은 각각의 Agent와 통신하며 관련 기능을 처리하고, Collector는 각각의 Agent와 통신하여 모니터링 메트릭스를 수집합니다. 본 내용에서는 Flamingo 2에서 제공하는 각종 Agent의 설치 방법을 알아봅니다.

Important
 Flamingo 2의 분산 파일 시스템 브라우저, 모니터링 부분은 기능은 특허가 적용되어 있습니다. 따라서 Flamingo의 소스코드 레파지토리에 제공하는 소스코드에서 Flamingo Agent는 소스코드를 제공하지 않으며 바이너만 제공하고 있습니다. 또한 제공한 Flamingo Agent의 decompile, modification 등은 특허에 위반될 수 있음을 알립니다.
Resource Manager Agent

Hadoop 2에서 추가된 Resource Manager 정보를 모니터링하고 YARN Application을 관리하기 위해서는 YARN Resource Manager용 Flamingo Agent를 설치해야 합니다. Cloudera CDH 배포판이 아닌 경우 YARN_OPTS 환경변수에 아래의 옵션을 추가해야 합니다.

-javaagent:flamingo2-hadoop2-rm-agent-2.0.0.jar=resourcescript:resourcemanager.bm

Cloudera CDH 배포판인 경우 Cloudera Manager에 로그안하여 Resource Manager 설정에서 ResourceManager의 Java 구성 옵션 에 아래와 같이 정보를 추가합니다.

JAR 파일의 경로는 반드시 절대 경로로 입력하도록 합니다.

-javaagent:/root/build/flamingo2/trunk/flamingo2-hadoop2-rm-agent/target/flamingo2-hadoop2-rm-agent-2.0.jar=resourcescript:resourcemanager.bm
-XX:+UseParNewGC -XX:+UseConcMarkSweepGC -XX:-CMSConcurrentMTEnabled
-XX:CMSInitiatingOccupancyFraction=70 -XX:+CMSParallelRemarkEnabled
Warning
 종종 JAR 파일에 접근할 수 있는 권한이 없는 경우 JVM이 정상적으로 동작하지 못하므로 만약 Resource Manager가 정상적으로 시작하지 않는다면 JAR 파일의 경로가 Resource Manager가 접근할 수 있는 권한을 가지고 있는지 확인하도록 합니다. Cloudera CDH, Pivotal HD, Hortonworks HDP 배포판은 Resource Manager의 시스템 계정은 yarn 이므로 Flamingo Resource Manager Agent JAR 파일이 yarn 계정으로 접근이 가능한 디렉토리에 있는지 확인하도록 합니다.
Cloudera CDH 문제 해결하기

Cloudera CDH의 경우 ResourceManager의 Java 구성 옵션 의 경로가 잘못된 경우 Resource Manager가 정상적으로 동작하지 않습니다. Flamingo Reesource Manager Agent를 ResourceManager의 Java 구성 옵션 에 설정한 후 다음과 같이 "yarn > 인스턴스"를 선택합니다. 그러면 다음과 같이 ResourceManager 항목을 확인할 수 있습니다.

ResourceManager 항목을 선택하면 Resource Manager에 대한 상세한 정보가 표시가 됩니다. 그러면 우측 메뉴에서 재시작을 선택합니다.

그러면 Resource Manager 프로세스만 재시작이 진행되며 약간의 시간이 소요됩니다. 만약 재시작중 실패하면 다음과 같이 실패 상태로 변경됩니다. 여기에서 원인을 찾기 위해서 Stdout을 선택합니다.

Resource Manager 구동 스크립트의 표준 출력이 로그 파일에 기록되어 나타나며 하단에 JVM을 초기화할 수 없다는 에러 문구를 확인할 수 있습니다.

이제 Stderr 로그를 확인해봅니다. 그러면 JAR, Zip 파일을 열 수 없다는 문구가 나타납니다. 보통 이 경우는 파일명 오류, 권한 오류, 파일 깨짐 등으로 인하여 파일을 열 수 없는 경우에 해당합니다. 따라서 이것을 점검하려면 yarn 계정으로 로그인하여 해당 파일이 정상적으로 접근이 되는지, 파일이 제대로 압축이 풀리는지 jar tvf <JAR 파일> 커맨드로 확인을 해야 합니다.

설정이 정상적으로 되었다면 Resource Manager를 재시작하면 다음의 화면을 확인할 수 있습니다.

Pivotal HAWQ JDBC Driver

R 연동

Flamingo는 Flamingo 2 부터 R과 함께 연동하여 사용할 수 있습니다. 연동을 위해서 R, RStudio, RHadop, RHive와 같은 오픈소스가 필요하게 됩니다. Flamingo는 RStudio Server와 연동하여 Flamingo를 로그인한 상태에서 RStudio Server와 연계하여 사용할 수 있습니다. 이렇게 되면 Flamingo에 로그인한 사용자가 Flamingo 내에서 R로 작업을 할 수 있게 됩니다. R은 대용량 분석은 아니지만 분석 알고리즘이 많고, 시각화 기능이 Hadoop에 비해서 상대적으로 매우 강력하기 때문에 R을 빅데이터 분석에서 같이 활용하는 것은 편의성과 생산성 등을 확보해주고 특히 분석 방법을 다양화시킬 수 있다는 장점을 얻게 됩니다.

R 설치

R을 Flamingo와 통합하기 위해서 우선 R을 사용할 수 있도록 제공하려는 서버에 R 패키지를 설치하는 과정이 우선적으로 필요합니다. 여기에서 Flamingo와 R을 같은 서버에서 제공하는 방법이 있겠지만 R을 이용하여 분석하는 경우 CPU, Memory, HDD를 소비하므로 원칙적으로 분리할 것을 권고합니다. 따라서 R을 설치한 서버, Flamingo를 설치한 서버 이렇게 두 대의 서버가 필요하게 됩니다. R은 Flamingo를 설치한 서버가 아닌 R을 이용하여 분석하는 서버에 설치를 해야 합니다.

R을 서버에 설치하기 위해서 root 로 로그인한 후 다음의 커맨드를 실행하여 R을 설치합니다(CentOS의 경우).

# yum install R

Ubuntu에서는 다음의 커맨드로 R을 설치할 수 있습니다.

# apt-get install r-base gdebi-core libapparmor1
Note
 R은 R 패키지를 인터넷을 통해서 다운로드하여 설치하도록 되어 있습니다. 따라서 별도 R 패키지 레파지토리를 구성하거나, R이 설치되어 있는 서버에 패키지를 설치 또는 구성해두지 않으면 인터넷 접속이 허용되지 않는 경우 R 패키지를 다운로드할 수 없습니다.

RStudio Server 설치

Flamingo는 RStudio Server와 연동하여 동작하도록 구현되어 있습니다. RStudio가 아닌 RStudio Server는 웹 브라우저를 통해서 R을 접근하여 사용할 수 있도록 해줍니다. 특히 Flamingo는 Flamingo의 사용자 계정과 RStudio Server의 사용자 계정을 통합할 수 있도록 해주어 하나의 사용자 체계를 가질 수 있도록 해줍니다. 다만 RStudio Server의 사용자 계정은 Linux 서버의 계정이며, Flamingo의 계정은 서비스 계정의 개념이므로 이를 통합하기 위해서는 조직내에서 몇 가지 고려해야할 점이 있습니다. RStudio Server를 통해서 접근하는 서버의 Linux 계정에 대한 생성의 권한을 부여할 것이냐 입니다. 여하튼 이 부분은 Flamingo와 RStudio Server를 통합하는 조직의 보안 정책이므로 이 문서에서는 다루지 않겠습니다.

우선 RStudio Server는 R을 설치한 서버에 설치해야 합니다. 이를 위해서 RStudio Server 다운로드를 합니다. 다운로드시 OS에 맞는 버전을 다운로드해야 합니다. 아래 설치 내용은 CentOS의 경우로 root 로 로그인하여 순서대로 진행합니다.

# yum install openssl098e # Required only for RedHat/CentOS 6 and 7
# wget http://download2.rstudio.org/rstudio-server-0.98.1103-x86_64.rpm
# yum install --nogpgcheck rstudio-server-0.98.1103-x86_64.rpm

Ubuntu에서는 apt-get 대신 gdebi 커맨드로 설치를 합니다. 아래 커맨드는 64비트 Ubuntu의 경우에 해당합니다.

# wget http://download2.rstudio.org/rstudio-server-0.98.1103-amd64.deb
# gdebi rstudio-server-0.98.1103-amd64.deb

RStudio Server 환경설정

RStudio Server는 기본으로 8787 포트를 사용합니다. 하지만 웹 브라우저에서 접근시 connection refused 가 발생하면 RStudio Server가 설치되어 있는 다음의 설정 파일을 확인하도록 합니다.

  • /etc/rstudio/rserver.conf 파일

  • /etc/rstudio/rsession.conf 파일

특히 RStudio Server의 포트 및 IP 주소는 /etc/rstudio/rserver.conf 파일에서 관리합니다. 네트워크 설정을 하기 위해서 이 파일에 다음을 추가합니다.

www-port=8787
www-address=192.168.0.1

이제 root 로 다음의 커맨드를 실행하여 RStudio Server를 재시작합니다.

# rstudio-server restart

정상적으로 포트가 열려있는지 확인하려면 다음의 커맨드를 실행하여 확인하도록 합니다.

# netstat -an | grep 8787
tcp        0      0 0.0.0.0:8787                0.0.0.0:*                   LISTEN

또한 /etc/rstudio/rserver.conf 파일에 다음을 추가하여 세션 만료 시간을 지정할 수 있습니다.

session-timeout-minutes=30
Note
 정책상 RStudio Server를 직접 외부로 포트를 개방할 수 없는 상황이지만 포워딩을 할 수 있는 상황이라면 rinetd 를 이용하여 포트를 포워딩하도록 시스템을 구축할 수 있습니다.

Flamingo의 RStudio Server 연동 설정

R과 RStudio Server를 설치하면 Flamingo와 RStudio Server를 연결하기 위해서 몇 가지 설정이 필요합니다.

  • Flamingo에서 RStudio Server에 접속할 정보 설정

  • RStudio Server에 Flamingo System Agent 설치 (root 권한 설치 필요)

RStudio Server 접속 정보 설정

Flamingo와 RStudio Server를 연결하기 위한 정보는 /WEB-INF/config.properties 파일에 있습니다. 이 파일에서 다음의 정보를 찾아서 IP 주소와 포트를 입력하도록 합니다.

rserver.auth.url=http://192.168.1.1:8787/auth-public-key
rserver.signin.url=http://192.168.1.1:8787/auth-do-sign-in
Flamingo System Agent 설치

RStudio Server가 Linux 사용자 계정을 사용하므로 Flamingo의 사용자와 RStudio Server의 사용자는 근본적으로 차이가 있습니다. 이 문제를 해결하기 위해서 Flamingo는 root 권한으로 RStudio Server에 System Agent를 설치하도록 합니다. Flamingo에 포함되어 있는 System Agent는 .war 파일 형식으로 배포되며 파일명은 flamingo2-system-agent-2.0.war 입니다. 따라서 사용자는 Apache Tomcat에 배포하여 동작시키면 됩니다. 다만 배포시 ROOT.war 로 파일명을 변경하고 기존의 <TOMCAT_HOME>/webapps/ROOT 디렉토리를 삭제한 후 배포하시기 바랍니다.

user.system.agent.urls=http://192.168.1.1:7070/remote/agent/system

RHive와 RHadoop 설치 참고사항

Hadoop의 HDFS에 저장되어 있는 파일을 R에서 로딩하여 분석하려면 HDFS와 연동하는 연동모듈 등이 필요합니다. RHive와 RHadoop은 연동을 위한 모듈로써 별도로 R, RStudio Server에 설치해야 합니다. RHive는 R에서 Hive QL을 실행하여 데이터를 R에서 사용할 수 있도록 하며, RHadoop의 rhdfs는 HDFS의 파일을 R에서 로딩할 수 있게 해줍니다.

  • RHive

  • RHadoop rhdfs

Note
 종종 많은 사용자들이 RHive, RHadoop(rhdfs)을 빅데이터 분석이 가능한 것으로 이해하고 있습니다. RHive, RHadoop(rhdfs)은 R 사용자를 위해서 Hadoop의 HDFS의 파일을 로딩할 수 있도록 하는 것이지 R을 고속 병렬 처리하는 개념이 아님을 알려드립니다. 또한 본 문서에서는 관련 내용만 전달하며 RHive, RHadoop을 Flamingo와 같이 사용하자고 하는 경우 별도 설치를 하거나 또는 기술지원(sales@cloudine.co.kr)을 요청하시기 바랍니다.
단일 계정으로 RStudio Server에 접근하고자 하는 경우

경우에 따라서 RStudio Server에 별도 프로세스가 사용자를 생성하거나 삭제하는 등의 작업을 보안상 허용하지 않을 수 있습니다. 이러한 이유로 하나의 단일 계정에서 R을 사용하고자 할 수도 있습니다. Flamingo는 기본으로 Flamingo 사용자과 RStudio Server가 연동되도록 구현되어 있기 때문에 이렇게 모든 사용자를 단일 사용자가 사용할 수 있도록 하기 위해서는 소스코드를 수정해서 적용해야 합니다. 따라서 이 부분에 대한 별도 지원이 필요한 경우 기술지원(sales@cloudine.co.kr)을 요청하거나 직접 수정후 GPL v3로 공개하시기 바랍니다.

Web Application Server 선택하기

Flamingo 2 부터는 WebSocket을 사용하므로 다음의 버전에 충족하도록 Web Application Server를 사용해야 합니다. 특히 Flamingo 2는 Apache Tomcat을 기반으로 개발이 되어 있으므로 Apache Tomcat을 사용하면 쉽게 구성할 수 있습니다. 다음은 Flamingo 2를 사용하기 위한 최소 요구조건에 충족하는 Web Application Server입니다.

  • Tomcat 7.0.47+

  • Jetty 9.1+

  • GlassFish 4.1+

  • WebLogic 12.1.3+

  • Undertow 1.0+ (WildFly 8.0+)

웹 다운로드 성능 향상 시키기

Flamingo를 사용하는 사용자의 웹 브라우저에서 다운로드하는 Tomcat의 <TOMCAT_HOME>/conf/server.xml 파일을 수정하여 사용자의 웹 브라우저에서 다운로드하는 스크립트의 용량을 줄일 수 있습니다. Tomcat의 Connector 부분을 찾아서 다음과 같이 압축 관련 옵션을 추가합니다.

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           compression="on"
           compressionMinSize="2048"
           noCompressionUserAgents="gozilla, traviata"
           compressableMimeType="text/css,text/html,text/xml,application/json,application/javascript,application/x-javascript,text/javascript,text/x-javascript,text/x-json"
           redirectPort="8443"/>
Note
 이 설정은 Apache Tomcat과 관련된 설정으로 다른 Web Application Server를 사용하는 경우 해당 벤더에 문의 하십시오.
UTF-8 설정하기

Flamingo는 UTF-8을 지원해야만 multi-bytes character를 정상적으로 처리할 수 있습니다. 이를 위해서 Tomcat의 커넥터에서 URI Encoding을 UTF-8을 사용하도록 설정해야 합니다. Tomcat의 <TOMCAT_HOME>/conf/server.xml 파일에서 다음과 같이 URIEncoding 옵션을 추가하도록 합니다.

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           URIEncoding="UTF-8"/>
JVM Heap 조정하기

Flamingo를 사용하는 서버는 MapReduce Job, Hive QL, Pig Latin 등을 실행하기 위해서 충분히 메모리를 확보해야 합니다. 하지만 Flamingo를 실행하는 Tomcat의 JVM Heap이 부족하게 설정이 되어 있다면 큰 의미가 없습니다. Tomcat의 JVM Heap을 설정하기 위해서 <TOMCAT_HOME>/bin/catalina.sh 파일의 앞쪽에 CATALINA_OPTS 변수 설정을 추가하도록 합니다.

#   LOGGING_MANAGER (Optional) Override Tomcat's logging manager
#                   Example (all one line)
#                   LOGGING_MANAGER="-Djava.util.logging.manager=org.apache.juli.ClassLoaderLogManager"
# -----------------------------------------------------------------------------

CATALINA_OPTS="-Dflamingo -Xms4G -Xmx4G"

# OS specific support.  $var _must_ be set to either true or false.
cygwin=false
darwin=false
os400=false
case "`uname`" in
CYGWIN*) cygwin=true;;
Darwin*) darwin=true;;
OS400*) os400=true;;
esac

MapReduce Job

MapReduce Job의 경우 모니터링 에이전트를 통해서 수집할 수 있는 것은 Job ID입니다. Job ID를 클라이언트측에서 수집함으로써 MapReduce Job을 추적할 수 있게 되며 Flamingo 2에서는 이 정보를 활용하여 워크플로우와 MapReduce Job의 연관 정보를 찾아냅니다. MapReduce Job의 Job ID를 추출하는 모니터링 에이전트는 다음을 대상으로 동작합니다.

  • MapReduce Job

  • Pig Latin

  • Hive QL

사용자가 커맨드 라인에서 MapReduce를 실행하거나, Pig Latin 스크립트를 실행하거나, Hive QL을 실행한다면 Job Tracker를 통해서 Job ID를 배정받습니다. 모니터링 에이전트는 이 Job ID를 별도로 기록하여 어떤 Job이 어떤 워크플로우에 해당하는지를 알아냅니다.

MapReduce Job의 Job ID 및 YARN Application의 ID를 추출하기 위해서 다음와 같이 HADOOP_CLIENT_OPTS 환경변수에 Flamingo 2 MapReduce Agent를 설정합니다. 이렇게 설정하면 MapReduce Job을 클라이언트에서 실행하는 경우 Job ID가 현재 디렉토리에 파일로 저장됩니다.

export HADOOP_CLIENT_OPTS=-javaagent:flamingo2-hadoop2-mr-agent-2.0.jar=resourcescript:mr2.bm

영구적으로 적용하고자 하는 경우 ~/.profile 또는 ~/.bashrc 파일에 설정할 수 있으며, <HADOOP_HOME>/bin/hadoop 커맨드에 환경변수를 설정할 수 있습니다.

워크플로우 개요

  • 다양한 분석 모듈 탑재 (예: MapReduce, Pig, Hive, Bash Shell, Python, Spark, R, Mahout, Ankus …​)

  • 분석 및 처리 모듈을 Drag-and-drop과 파라메터 입력으로 손쉽게 처리

  • 복잡한 데이터 추출, 가공 및 적재 과정을 처리하는 순서를 신속하게 설계

  • 다양한 알고리즘을 통합하여 적절한 데이터 분석 활동의 편의를 제공

    • 워크플로우 디자이너를 활용한 알고리즘 제공 및 플랫폼 통합

디렉토리 업로드

트리 경로를 선택한 후 마우스 우클릭 시 나타나는 메뉴에서 업로드를 선택합니다.

Note
 단일 파일 기준 업로드 가능한 최대 용량은 기본값으로 100MB 이내로 정해져 있습니다. 이 값을 변경하기 위해서는 config.properties 파일의 file.upload.max.size 값을 변경해야 합니다.

업로드 방법은 두 가지 방식을 제공합니다.

  • 찾기 버튼을 선택 후 업로드 할 파일을 선택하거나 드래그 앤 드롭해서 업로드 창에 추가합니다.

  • OS에서 제공되는 파일 브라우저에서 직접 파일을 드래그 앤 드롭해서 업로드 창에 추가합니다.

업로드 할 파일 목록을 확인 후 업로드 버튼을 선택합니다.

업로드가 완료되면 상태 메시지가 완료로 변경됩니다.

업로드 창을 닫으면 선택한 경로가 갱신되고 업로드한 파일 목록이 보여집니다.

디렉토리 병합

트리 경로를 선택한 후 마우스 우클릭 시 나타나는 메뉴에서 병합을 선택합니다. 파일 병합은 선택한 경로에 존재하는 모든 파일을 하나의 파일로 병합하게 됩니다.

병합할 기본 파일명은 트리에서 선택한 디렉토리명으로 지정되어 있기 때문에 반드시 파일명을 변경해야 합니다.

병합할 새로운 파일명을 입력합니다.

병합할 파일의 저장 위치는 선택한 디렉토리의 상위 경로에 저장됩니다. 만약 루트 경로에 있는 파일 들을 병합할 경우 병합된 파일의 저장위치는 루트 경로에 저장됩니다.

상위 경로로 이동 후 병합된 파일을 확인합니다.

병합된 파일을 더블클릭하면 파일 속성 정보 창을 통해 파일 정보를 확인할 수 있습니다.

디렉토리 속성 정보 창에서 선택한 디렉토리 정보를 확인할 수 있습니다.

디렉토리 갱신

디렉토리를 갱신하는 방법을 두 가지 방식을 제공합니다.

  • 선택한 디렉토리의 하위 디렉토리 및 파일 목록이 갱신됩니다.

  • 루트 경로를 기준으로 전체 디렉토리 및 파일 목록이 갱신됩니다.

디렉토리 권한 설정

트리 경로를 선택한 후 마우스 우클릭 시 나타나는 메뉴에서 권한을 선택합니다.

접근 권한 설정은 선택한 디렉토리를 포함한 모든 하위 디렉토리 및 파일 들의 소유권 및 권한을 변경할 수 있습니다.

HAWQ 설정

HAWQ 접속을 위해서는 접속형태, Database, User, CIDR-주소, 인증방법을
$MASTER_DATA_DIRECTORY/pg_hba.conf 파일 하단부에 추가해줘야 합니다.

$ cd $MASTER_DATA_DIRECTORY
$ vi pg_hba.conf
host  all  gpadmin  172.16.213.1/28  trust
$ gpstop -u(시스템에 반영)

Flamingo2 설정

Flamingo2 에서 HAWQ 접속을 위해
$TOMCAT_DIRECTORY/webapps/ROOT/WEB-INF/hadoop.properties 파일 설정을 해줍니다.

###########################################
## Pivotal HAWQ Configuration
## hawq.jdbc.type={greenplum|postgresql}
###########################################
default.hawq.jdbc.type=greenplum
default.hawq.greenplum.connectionUrl=jdbc:pivotal:greenplum://
default.hawq.postgresql.connectionUrl=jdbc:postgresql://
default.hawq.host=172.16.213.138
default.hawq.port=5432
default.hawq.databaseName=gpadmin
default.hawq.user=gpadmin
default.hawq.password=
default.hawq.autoCommit=false
default.hawq.driver=com.pivotal.jdbc.GreenplumDriver
default.hawq.postgresql.driver=org.postgresql.Driver

여기서 설정해줘야 할 값들은

default.hawq.jdbc.type
default.hawq.host
default.hawq.port
default.hawq.databaseName
default.hawq.user
default.hawq.password

입니다.

default.hawq.jdbc.type 은 greenplum 과 postgresql 둘 중에서 선택 가능합니다.
greenplum 으로 설정하면 JDBC driver 가 greenplum 으로 로딩되고
postgresql 로 설정하면 JDBC driver 가 postgres 로 로딩됩니다.
greenplum 으로 설정시 반드시 WEB-INF/lib/greenplum.jar 파일이 있어야 합니다.

설정이 완료되면 tomcat 을 재시작 하고 HAWQ 에 접속합니다.

HDFS 브라우저 권한 관리

Flamingo2는 HDFS 브라우저 권한을 관리하기 위해 다음의 기능 들을 제공합니다.

Note
 HDFS 브라우저 권한 관리를 포함한 시스템 관리 메뉴는 관리자 권한의 계정에서만 사용할 수 있습니다.

권한

HDFS 브라우저 권한 관리는 Flamingo2의 HDFS 브라우저에서 제공되는 기능 들(디렉토리 생성/삭제, 파일 삭제/다운로드 등)을 사용자 등급별로 사용 권한을 설정할 수 있습니다. 사용자 등급은 관리자(1등급), 사용자(2~5등급)에 따라 구분되며, 관리자(admin) 계정은 HDFS 브라우저의 모든 경로에서 브라우저 기능 사용 권한을 가집니다. 일반 사용자의 경우 사용자 관리 메뉴에서 관리자의 회원가입 승인 처리가 완료되는 시점에서 자동으로 지정된 HDFS 사용자 홈 디렉토리(default=/user) 하위 경로에 권한 정보가 추가됩니다. 따라서 관리자를 제외한 모든 일반 사용자는 HDFS 파일 시스템 상에서 자신의 홈 디렉토리(ex. /user/cloudine/**)로 시작되는 경로에서만 브라우저 기능을 사용할 수 있습니다.

Note
 HDFS 파일 시스템의 사용자 홈 디렉토리 기본 경로를 변경하려면 config.properties의 user.home.hdfs.path 값을 변경해야 합니다.

권한 추가

HDFS 파일시스템에서 권한 추가 버튼을 클릭합니다.

선택한 경로는 HDFS 파일 시스템 경로와 동일하며, 전체 경로 혹은 세부 경로에 대한 권한을 설정할 수 있습니다.

선택한 경로의 하위 모든 경로까지 권한 정보를 적용하려면 하위경로포함 옵션(/**)을 체크합니다.

선택한 경로에 대한 권한 정보가 추가되고, 추가한 권한 정보를 선택하면 상세 권한 정보를 확인할 수 있습니다.

권한 삭제

삭제할 권한을 선택 후 삭제 버튼을 클릭합니다.

삭제할 권한 정보를 확인 후 삭제합니다.

권한 수정

변경할 권한 정보를 선택 후 권한 수정 버튼을 클릭합니다.

사용자 승인 후에 추가되는 권한은 각 사용자 홈 디렉토리 하위 모든 경로에 대해서 디렉토리 및 파일 사용 권한이 부여됩니다.

경로, 권한, 등급의 경우 수정을 원하는 부부의 체크 박스를 선택해서 수정합니다. 디렉토리 및 파일의 경우 사용 금지을 원하는 기능을 체크 해제한 후 저장 버튼을 클릭하면 선택하 경로에 대한 권한이 업데이트 됩니다. 만약 하위 디렉토리를 포함하지 않고 특정 경로에서만 사용 권한을 부여하고 싶을 경우 경로 뒤의 "/**" 문자를 제거해야 합니다(ex. /user/cloudine).

수정된 권한 정보를 선택하면 HDFS 브라우저 사용 권한 정보 필드에 상세 정보가 보여집니다.
